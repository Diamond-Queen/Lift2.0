# Stripe Setup\n\nThis project includes placeholders for Stripe integration. In development, checkout fields are disabled and mocked flows may be used. To enable real payments, configure Stripe as below.\n\n## Prerequisites\n- Stripe account: https://dashboard.stripe.com\n- Node.js and npm installed\n- Secrets added to `.env.local`\n\n## Required Environment Variables\nAdd the following to `.env.local` (see `.env.example`):\n\n- `STRIPE_SECRET_KEY` — Your Stripe secret key (`sk_test_...` or `sk_live_...`)\n- `STRIPE_PUBLISHABLE_KEY` — Your publishable key (`pk_test_...` or `pk_live_...`)\n- `STRIPE_WEBHOOK_SECRET` — From Stripe CLI or Dashboard for your webhook endpoint\n- `STRIPE_PRICE_CAREER` — Price ID for the Career plan\n- `STRIPE_PRICE_NOTES` — Price ID for the Notes plan\n- `STRIPE_PRICE_FULL` — Price ID for the Full Access plan\n- `NEXTAUTH_URL` — Base URL of the app (e.g., `http://localhost:3000`)\n\nOptional dev flags:\n- `STRIPE_DEV_MODE=true` — Server-side dev mode\n- `NEXT_PUBLIC_STRIPE_DEV_MODE=true` — Client-side dev mode\n\n## Webhook (local)\nInstall Stripe CLI and forward events to your local app:\n\n```bash\nstripe login\nstripe listen --forward-to localhost:3000/api/subscription/webhook\n```\n\n## Notes\n- Never commit secrets. `.env.local` is ignored by git.\n- Client-side card fields are intentionally disabled until Stripe Elements is integrated.\n- See `pages/api/subscription/checkout.js` and `pages/api/subscription/webhook.js` for the integration points.\n
